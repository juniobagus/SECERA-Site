# Panduan Upload Asset CMS (Untuk Marketing/Admin)

## Tujuan
Biar foto dan video di website tetap tajam di HP, tapi tetap cepat dibuka.

## Aturan Singkat yang Wajib Diikuti
1. Jangan upload file dari WhatsApp (biasanya kualitas turun).
2. Gunakan file original dari fotografer/designer.
3. Untuk foto hero banner: lebar minimal 1600px.
4. Untuk foto produk: lebar minimal 1200px.
5. Untuk video: minimal kualitas 720p.

## Checklist Sebelum Upload
1. Cek foto tidak blur saat di-zoom 100% di laptop.
2. Cek video tidak patah-patah dan suaranya bersih.
3. Nama file jelas, contoh: `secera-ramadhan-hero-01.jpg`.
4. Pastikan rasio gambar sesuai slot (banner jangan terlalu portrait).

## Langkah Upload di CMS
1. Masuk ke halaman CMS yang ingin diubah (Home/Shop/Product).
2. Upload file master (versi terbaik/original).
3. Tunggu status proses sampai `Ready`.
4. Lihat preview mobile di CMS.
5. Kalau preview sudah pas, baru klik Publish.

## Arti Status di CMS
1. `Processing`: sistem sedang membuat versi ringan + tajam otomatis.
2. `Ready`: asset sudah siap tampil optimal di website.
3. `Degraded Ready`: sebagian format gagal dibuat, tapi asset masih bisa tampil.
4. `Failed`: upload gagal diproses, perlu upload ulang.

## Jika Asset Ditolak Sistem
Penyebab umum:
1. Resolusi terlalu kecil.
2. Format file tidak didukung.
3. File rusak/corrupt.

Yang harus dilakukan:
1. Ambil ulang file original.
2. Cek ukuran pixel sesuai minimum.
3. Upload ulang.

## Quality Check Setelah Publish (Wajib)
1. Buka halaman dari HP (bukan hanya desktop).
2. Pastikan gambar hero dan produk terlihat tajam.
3. Cek halaman tetap cepat dibuka.
4. Cek video muncul poster dulu sebelum play.

## Do and Don't
Do:
1. Simpan master file di folder tim yang rapi.
2. Gunakan naming file konsisten per campaign.
3. Selalu preview mobile sebelum publish.

Don't:
1. Jangan screenshot lalu upload ulang screenshot.
2. Jangan upload file yang sudah berkali-kali di-compress.
3. Jangan publish saat status masih `Processing`.

## SLA Internal (Disarankan)
1. Upload normal: tunggu maksimal 2-5 menit sampai `Ready`.
2. Jika lebih dari 10 menit masih `Processing`, lapor tim tech.
3. Jika `Failed` 2x berturut-turut, kirim file ke tim tech untuk pengecekan.
